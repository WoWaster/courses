### CSV pretty printer

Требуется разработать консольную утилиту, преобразующую CSV файл (https://en.wikipedia.org/wiki/Comma-separated_values) `input.csv` в таблицу в формате псевдографики и записывающую её в файл `output.txt`.

Правила отрисовки:
- Заголовок должен быть выделен. Поля заголовка выравнены влево.
- Данные: текст — выравнен влево, числа — выравнены вправо.
- Ширина столбцов одинаковая и соответствует максимальному элементу столбца.

Пример выходного файла:
```
+===================+==============+
| Test field 1      | Test field 2 |
+===================+==============+
| test              |          123 |
+-------------------+--------------+
| long string test! |         28.7 |
+-------------------+--------------+
| other text        |            3 |
+-------------------+--------------+
```

Обратите внимание, __НЕ__ нужно поддерживать экранированные строки и многобайтовые кодировки.

Будет полезно изучить спецификаторы `printf`-подобных функций и, в частности, функцию `sprintf`.
(см. https://en.cppreference.com/w/c/io/fprintf.html, `man 3 printf` и `man 3 sprintf`)

### Работа в классе

#### Задача A — Структура и вставка

Опишите абстрактный тип данных — двоичное дерево поиска (`BST`) — и его структуру.

Реализуйте функции:
- `void bstInsert(BST* tree, int value)` — вставить значение в дерево; если значение уже есть, ничего не делать.
- `bool bstContains(BST* tree, int value)` — вернуть `true`, если значение есть в дереве.
- `void bstFree(BST* tree)` — освободить всю память, занятую деревом.

Напишите `main` с несколькими простыми проверками: вставьте несколько элементов и убедитесь, что `bstContains` возвращает правильные результаты.

#### Задача B — Обходы

Реализуйте функции обхода дерева, каждая из которых печатает элементы в соответствующем порядке. Все функции должны быть реализованы **рекурсивно**:
- `void bstInorder(BST* tree)`
- `void bstPreorder(BST* tree)`
- `void bstPostorder(BST* tree)`

Все функции должны корректно работать на пустом дереве.

#### Задача C — Статистика дерева

Реализуйте функции:
- `int bstHeight(BST* tree)` — высота дерева (пустое дерево имеет высоту 0).
- `int bstSize(BST* tree)` — количество узлов.
- `int bstMin(BST* tree)` — минимальное значение.
- `int bstMax(BST* tree)` — максимальное значение.

Для `bstMin` и `bstMax` определите поведение на пустом дереве и задокументируйте его в заголовочном файле.

### ДЗ № 2. Бинарное дерево поиска

#### Задача D — Удаление

Реализуйте функцию:
- `void bstDelete(BST* tree, int value)` — удалить узел с заданным значением.

Функция должна корректно обрабатывать все три случая:
- узел является листом,
- узел имеет одного потомка,
- узел имеет двух потомков.

Если значения в дереве нет, функция не должна падать или изменять дерево.

#### Задача E — Слияние двух деревьев

Реализуйте функцию:
- `BST* bstMerge(BST* tree1, BST* tree2)` — вернуть новое дерево, содержащее все элементы из обоих деревьев без дубликатов.

Исходные деревья не должны изменяться.

#### Задача F — Проверка корректности

Реализуйте функцию:
- `bool bstIsValid(BST* tree)` — вернуть `true`, если дерево является корректным BST.

Обратите внимание: проверка только соседних узлов (`left < root < right`) недостаточна — подумайте, почему, и реализуйте корректное решение.

#### Задача G — k-й минимальный элемент

Реализуйте функцию:
- `int bstKthMin(BST* tree, int k)` — вернуть k-й минимальный элемент дерева (k начинается с 1).

Определите поведение при некорректном `k` (например, `k <= 0` или `k > bstSize(tree)`) и задокументируйте его в заголовочном файле.

#### Задача H — Итератор

Реализуйте итератор, который обходит дерево в порядке `inorder` **без рекурсии** — используя дополнительные структуры данных.

Интерфейс:
```c
Iterator* iteratorInit(BST* tree);
bool iteratorHasNext(Iterator* it);
int iteratorNext(Iterator* it);
void iteratorFree(Iterator* it);
```

Итератор не должен изменять дерево. Поведение при вызове `iteratorNext` на исчерпанном итераторе определите сами и задокументируйте.

## ДЗ № 3. АВЛ-дерево — «Регистрация рейсов»

> Claude Sonnet 4.6 с редактурой @WoWaster

Вы — стажёр в небольшой авиационной IT-компании **AeroSoft**.
Компания разрабатывает систему регистрации пассажиров для региональных аэропортов.
Система получает от партнёров сообщения с трёхбуквенными кодами IATA (`SVO`, `JFK`, `LED`...) и должна моментально отображать пассажиру полное название аэропорта назначения.

Предыдущий разработчик, Антон, хранил все данные в односвязном списке.
Пока в базе было 200 аэропортов — всё работало.
Но партнёры прислали полный датасет: **9 100 записей**.
Поиск в списке занимает секунды, на стойке регистрации выстраиваются очереди, директор недоволен.

Антон уволился.
Теперь это ваша проблема.

Вы открываете ноутбук и пишете: *«Нам нужно АВЛ-дерево»*.

### Датасет

Скачайте исходный CSV с данными об аэропортах:

```
https://raw.githubusercontent.com/datasets/airport-codes/master/data/airport-codes.csv
```

Из него нужно извлечь колонки `iata_code` и `name`, пропустив записи с пустым IATA-кодом.
Для этого воспользуйтесь прилагаемым скриптом `prepare.py`:

```python
import csv

with open("airport-codes.csv", encoding="utf-8") as fin, \
     open("airports.txt", "w", encoding="utf-8") as fout:
    reader = csv.DictReader(fin)
    for row in reader:
        code = row["iata_code"].strip()
        name = row["name"].strip()
        if code:
            fout.write(f"{code}:{name}\n")
```

Результирующий файл `airports.txt` выглядит так:

```
00A:Total Rf Heliport
00AK:Lowell Field
AAA:Anaa Airport
AMS:Amsterdam Airport Schiphol
JFK:John F Kennedy International Airport
LED:Pulkovo Airport
SVO:Sheremetyevo International Airport
...
```

Этот файл является входными данными для вашей программы.

### Задание

Реализуйте на языке C программу, которая загружает базу аэропортов в АВЛ-дерево и предоставляет интерфейс командной строки для работы с ней.

### Команды

| Команда                | Действие                                                  |
| ---------------------- | --------------------------------------------------------- |
| `find <код>`           | Найти название аэропорта по коду IATA                     |
| `add <код>:<название>` | Добавить новый аэропорт в базу                            |
| `delete <код>`         | Удалить аэропорт из базы                                  |
| `save`                 | Сохранить текущее состояние базы обратно в `airports.txt` |
| `quit`                 | Завершить работу программы                                |

### Пример сессии

```
$ ./aerosoft airports.txt
Загружено 9096 аэропортов. Система готова к работе.

> find SVO
SVO → Sheremetyevo International Airport

> find ZZZ
Аэропорт с кодом 'ZZZ' не найден в базе.

> add ZZZ:Nowhere International Airport
Аэропорт 'ZZZ' добавлен в базу.

> find ZZZ
ZZZ → Nowhere International Airport

> delete ZZZ
Аэропорт 'ZZZ' удалён из базы.

> save
База сохранена: 9096 аэропортов.

> quit
```

### Бонусное задание: «Антон был не так уж неправ?» (+5 баллов)

Перед тем как окончательно списать код Антона в утиль, руководство хочет увидеть **цифры**.
Директор по разработке говорит: *«Ты утверждаешь, что АВЛ-дерево быстрее. Докажи. У нас production, нам нужны данные, а не слова»*.

Вам предстоит провести честное сравнение двух реализаций и представить отчёт.

#### Что нужно сделать

Реализуйте альтернативную версию словаря на **односвязном списке** с тем же интерфейсом, что и АВЛ-дерево — функции `insert`, `find`, `delete`.

Реализуйте два сценария нагрузки, имитирующих реальную работу системы регистрации:

**Сценарий 1 — «Утренний час пик»:** 50 000 операций `find` по случайным кодам из датасета.

**Сценарий 2 — «Обновление базы»:** 10 000 операций `insert` с новыми кодами, перемежающихся с 10 000 операций `find`.

Каждый сценарий запускается отдельно для списка и для АВЛ-дерева.

#### Профилирование с помощью perf

Замерьте производительность каждого сценария с помощью `perf stat`.

Для более глубокого анализа постройте профиль горячих функций с помощью `perf record`.

Посмотрите, какие функции списка потребляют больше всего времени. Ожидаемо ли это?

#### Отчёт

Напишите отчёт со следующими разделами:

**1. Таблица результатов** — время выполнения каждого сценария для обеих структур (возьмите значение `elapsed` из вывода `perf stat`).

**2. Анализ `perf report` для связного списка** — какие функции оказались в топе, почему именно они.

**3. Объяснение разницы** — почему АВЛ-дерево выигрывает у списка на операции `find`? Какова асимптотика каждой операции для обеих структур? Меняется ли соотношение для `insert`?

**4. Вывод для директора** — два-три предложения в деловом стиле, которые вы могли бы вставить в реальный внутренний отчёт компании AeroSoft. Без формул, только по делу.
