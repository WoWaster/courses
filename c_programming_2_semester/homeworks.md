### Настройка репозитория

В прошлом семестре Вы познакомились с инструментами разработчика на Си.
Сейчас придётся использовать все полученные знания.

Обязательно нужно.
1. Создать новый репозиторий.
2. Настроить в нём CI с clang-format __И__ clang-tidy (пример по здесь: https://github.com/WoWaster/spbu-c-ci-example).
3. Использовать CMake.
4. Писать тесты к своему коду.
5. Не забыть про лицензию и хоть какой-нибудь README.md.

Если что-то забылось, всегда можно обратиться к материалам прошлого семестра: https://github.com/WoWaster/courses/tree/main/c_programming

Можно:
1. Подготовить репозиторий самостоятельно без отдельного PR.
2. Переиспользовать библиотеки из прошлого семестра.
3. Торговаться с преподавателем на тему отключения *странных* правил clang-tidy.

__ВАЖНО__: при отсутствии "зеленой галочки" велик шанс получить 0 баллов за задачу сразу же, без какой-либо проверки.


### CSV pretty printer

Требуется разработать консольную утилиту, преобразующую CSV файл (https://en.wikipedia.org/wiki/Comma-separated_values) `input.csv` в таблицу в формате псевдографики и записывающий её в файл `output.txt`.

Правила отрисовки:
- Заголовок должен быть выделен. Поля заголовка выравнены влево.
- Данные: текст — выравнен влево, числа — выравнены вправо.
- Ширина столбцов одинаковая и соответствует максимальному элементу столбца.

Пример выходного файла:
```
+===================+==============+
| Test field 1      | Test field 2 |
+===================+==============+
| test              |          123 |
+-------------------+--------------+
| long string test! |         28.7 |
+-------------------+--------------+
| other text        |            3 |
+-------------------+--------------+
```

Обратите внимание, __НЕ__ нужно поддерживать экранированные строки и многобайтовые кодировки.

Будет полезно изучить спецификаторы `printf`-подобных функций и, в частности, функцию `sprintf`.
(см. https://en.cppreference.com/w/c/io/fprintf.html, `man 3 printf` и `man 3 sprintf`)
